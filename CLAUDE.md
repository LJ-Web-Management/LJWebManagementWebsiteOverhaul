# Instructions for working in this repo

## Never use em dashes ("—")

Do not use the em dash character (—) anywhere in this repo: page copy,
JSON-LD/structured data, code comments, commit messages, or
documentation. This applies to every file, not just rendered HTML.

Write the sentence a different way instead:
- Two related clauses: use a comma, semicolon, or colon depending on how
  they relate (`", "`, `"; "`, `": "`).
- A short interrupting aside: use parentheses, e.g. `"the tools you
  already use (CRMs, email platforms, and more)"`.
- Two independent sentences: just end the first one with a period and
  start a new sentence.

This rule was added after a sitewide sweep to remove every existing em
dash from the site's HTML, JSON-LD schema, CSS comments, and
documentation, so the codebase is currently clean. Keep it that way.
